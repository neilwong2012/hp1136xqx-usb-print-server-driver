# HP LaserJet M1136 USB Print Server XQX

macOS CUPS driver package for printing to an HP LaserJet Professional M1136 MFP
through a USB print server that provides raw TCP socket pass-through, such as
FusionWrt/OpenWrt `p910nd`.

中文说明见下方。English documentation follows the Chinese section.

## 中文说明

### 这是什么

这是一个给 macOS 使用的 CUPS 打印驱动安装包，用于让 HP LaserJet
Professional M1136 MFP 通过支持 raw TCP socket 透传的 USB 打印服务器打印，
例如 FusionWrt/OpenWrt 路由器上的 `p910nd`。

它不是 HP 官方驱动，也不是 AirPrint 驱动。它使用 `foo2xqx` 把 macOS 产生的
PDF/PostScript 打印任务转换成 HP M1136 这类 host-based 打印机能识别的数据。

### 使用条件

满足下面条件时才适合使用：

- 打印机是 HP LaserJet Professional M1136 MFP，或非常接近的 `foo2xqx`/XQX
  host-based HP LaserJet 机型。
- 打印机通过 USB 接入打印服务器。
- USB 打印服务器提供 raw TCP socket / JetDirect 透传服务，例如
  FusionWrt/OpenWrt `p910nd`。
- 打印服务器不会改写、转换或压缩打印数据。
- Mac 可以访问打印服务器的打印端口，例如：

```sh
nc -zv 192.168.31.119 9100
```

如果显示连接成功，说明网络和打印服务端口可用。

USB 打印服务器侧通常需要：

- USB 打印机已被系统识别，例如 VID/PID 类似 `03f0/042a`。
- USB 打印服务器已启用。
- 监听端口通常是 `9100`。
- 监听地址应绑定到 Mac 可访问的网络接口；在 Linux/FusionWrt/OpenWrt 上通常是
  LAN 或 `0.0.0.0`。

### 不适用场景

下面场景不适合使用本项目：

- 需要扫描功能。`p910nd` 只共享打印，不共享扫描。
- 想通过 AirPrint、IPP Everywhere 或 Bonjour 自动发现打印。
- USB 打印服务器不是 raw socket 透传模式，或者会改写打印数据。
- 打印机本身支持标准 PCL/PostScript，并且 Generic PCL/PostScript 已经能正常工作。
- 想安装 Apple/HP 官方签名驱动。
- 不愿意修改 CUPS sandbox 配置。本安装包为了兼容旧式 CUPS filter，会备份
  `/private/etc/cups/cups-files.conf` 并设置 `Sandboxing relaxed`。

### 安装

从 GitHub Releases 下载最新版 `.pkg` 并安装：

https://github.com/neilwong2012/hp1136xqx-usb-print-server-driver/releases

默认安装后会创建：

- 队列名：`HP1136XQX`
- 默认地址：`socket://192.168.31.119:9100`
- 纸张：A4
- 分辨率：600 dpi

安装包目前未签名、未公证。macOS 可能会提示无法确认开发者，需要在系统设置里手动允许。

### 配置路由器地址和端口

安装后可以随时修改路由器地址和端口：

```sh
sudo hp1136xqx-setup --host 192.168.31.119 --port 9100
```

如果路由器 IP 是其他地址：

```sh
sudo hp1136xqx-setup --host 192.168.1.1 --port 9100
```

也可以自定义 CUPS 队列名：

```sh
sudo hp1136xqx-setup --host 192.168.1.1 --port 9100 --queue HP1136Office
```

### 测试

检查端口：

```sh
nc -zv 192.168.31.119 9100
```

检查 CUPS 队列：

```sh
lpstat -p HP1136XQX -v HP1136XQX
```

打印简单测试：

```sh
echo test | lp -d HP1136XQX
```

### 图片或扫描件打印发黑

微信图片、拍照扫描件、灰底截图如果直接打印出来发黑，通常不是路由器或
`p910nd` 的问题，而是灰色纸面背景被 1-bit 激光打印半色调转成了密集黑点。

推荐先安装 ImageMagick：

```sh
brew install imagemagick
```

然后用安装包附带的图片预处理命令打印：

```sh
hp1136xqx-print-image /path/to/image.jpg
```

这个命令会清理灰底、转成黑白、按 A4 300dpi 生成 PDF，再发送到
`HP1136XQX` 队列。默认使用 `--fill` 铺满 A4，比例会保持正常，但边缘可能会被
裁切；M1136 这类激光打印机本身不能真正无边距打印，纸边仍会保留硬件不可打印区。

如果希望完整保留图片内容、接受纸边留白：

```sh
hp1136xqx-print-image --fit /path/to/image.jpg
```

只生成清理后的 PDF、不立刻打印：

```sh
hp1136xqx-print-image --no-print --output cleaned-a4.pdf /path/to/image.jpg
```

如果文字仍偏浅或背景仍偏黑，可以微调阈值：

```sh
hp1136xqx-print-image --white-point 86% --threshold 72% /path/to/image.jpg
```

对于普通 PDF 或文本作业，如果只是整体墨量偏重，可以使用浅色省墨模式：

```sh
lp -d HP1136XQX -o HPTonerMode=Light image.jpg
```

如果希望这个队列默认都用浅色模式：

```sh
sudo lpadmin -p HP1136XQX -o HPTonerMode=Light
```

恢复普通模式：

```sh
sudo lpadmin -p HP1136XQX -o HPTonerMode=Normal
```

### 从源码构建

需要 Homebrew 依赖：

```sh
brew install ghostscript gnu-sed
```

如果要使用 `hp1136xqx-print-image` 图片预处理命令，还需要：

```sh
brew install imagemagick
```

构建安装包：

```sh
./scripts/build-pkg.sh
```

生成的 `.pkg` 会放在 `dist/`。

### 卸载

```sh
sudo hp1136xqx-uninstall
```

卸载脚本会删除队列和本项目安装的文件。它不会自动恢复 CUPS sandbox 配置，因为安装器会备份原配置，而且一台机器上可能安装过多个版本。备份文件位于：

```text
/private/etc/cups/cups-files.conf.hp1136xqx-backup-*
```

## English

### What This Is

This is a macOS CUPS driver package for printing to an HP LaserJet Professional
M1136 MFP through a USB print server that provides raw TCP socket pass-through,
such as FusionWrt/OpenWrt `p910nd`.

It is not an official HP driver and it is not an AirPrint driver. It uses
`foo2xqx` to convert PDF/PostScript jobs generated by macOS into the XQX data
format expected by HP host-based printers such as the M1136.

### Requirements

Use this package only when these conditions are true:

- The printer is an HP LaserJet Professional M1136 MFP, or a very similar
  `foo2xqx`/XQX host-based HP LaserJet model.
- The printer is connected by USB to a print server.
- The USB print server provides raw TCP socket / JetDirect pass-through, such
  as FusionWrt/OpenWrt `p910nd`.
- The print server does not rewrite, convert, or compress the print data.
- The Mac can reach the print server's raw socket port, for example:

```sh
nc -zv 192.168.31.119 9100
```

On the USB print-server side, the usual requirements are:

- The USB printer is detected, often with an HP VID/PID such as `03f0/042a`.
- The USB print server is enabled.
- The listening port is usually `9100`.
- The service is bound to a network interface reachable from the Mac; on
  Linux/FusionWrt/OpenWrt this is usually the LAN interface or `0.0.0.0`.

### Not Supported

This project is not suitable for:

- Scanning. `p910nd` shares printing only; it does not share the scanner.
- AirPrint, IPP Everywhere, or Bonjour auto discovery.
- USB print servers that are not raw socket pass-through, or that rewrite print
  data.
- Printers that already work correctly with Generic PCL or Generic PostScript.
- Users who need an Apple/HP signed official driver.
- Environments that cannot relax CUPS sandboxing. For compatibility with this
  legacy CUPS filter, the installer backs up
  `/private/etc/cups/cups-files.conf` and sets `Sandboxing relaxed`.

### Install

Download the latest `.pkg` from GitHub Releases:

https://github.com/neilwong2012/hp1136xqx-usb-print-server-driver/releases

By default, the installer creates:

- Queue: `HP1136XQX`
- Address: `socket://192.168.31.119:9100`
- Paper: A4
- Resolution: 600 dpi

The package is currently unsigned and not notarized. macOS may require manual
confirmation in System Settings before installation.

### Configure Router Address And Port

After installation, run:

```sh
sudo hp1136xqx-setup --host 192.168.31.119 --port 9100
```

For a different router:

```sh
sudo hp1136xqx-setup --host 192.168.1.1 --port 9100
```

You can also change the queue name:

```sh
sudo hp1136xqx-setup --host 192.168.1.1 --port 9100 --queue HP1136Office
```

### Test

Check the socket:

```sh
nc -zv 192.168.31.119 9100
```

Check the CUPS queue:

```sh
lpstat -p HP1136XQX -v HP1136XQX
```

Print a small test:

```sh
echo test | lp -d HP1136XQX
```

### Dark Photo Or Scan Output

For WeChat images, photographed documents, scans, or gray-background screenshots
that print too dark, the router and `p910nd` are usually not the problem. The
gray paper background is being converted into dense 1-bit laser halftone dots.

Install ImageMagick first:

```sh
brew install imagemagick
```

Then print through the bundled image preprocessor:

```sh
hp1136xqx-print-image /path/to/image.jpg
```

This command cleans the gray background, converts the image to black and white,
places it on an A4 300 dpi page, generates a PDF, and sends it to the
`HP1136XQX` queue. It uses `--fill` by default, which fills A4 while preserving
proportions and may crop the edges. The M1136 cannot do true borderless printing,
so the printer's hardware margins still apply.

To preserve the whole image and allow page margins:

```sh
hp1136xqx-print-image --fit /path/to/image.jpg
```

To generate the cleaned PDF without printing:

```sh
hp1136xqx-print-image --no-print --output cleaned-a4.pdf /path/to/image.jpg
```

If the text is too light or the background is still too dark, adjust the
threshold:

```sh
hp1136xqx-print-image --white-point 86% --threshold 72% /path/to/image.jpg
```

For normal PDFs or text jobs that only need lower toner density, use light toner
mode:

```sh
lp -d HP1136XQX -o HPTonerMode=Light image.jpg
```

To make light mode the queue default:

```sh
sudo lpadmin -p HP1136XQX -o HPTonerMode=Light
```

To restore normal mode:

```sh
sudo lpadmin -p HP1136XQX -o HPTonerMode=Normal
```

### Build From Source

Requirements:

```sh
brew install ghostscript gnu-sed
```

For the optional `hp1136xqx-print-image` helper:

```sh
brew install imagemagick
```

Build:

```sh
./scripts/build-pkg.sh
```

The installer will be written to `dist/`.

### Uninstall

```sh
sudo hp1136xqx-uninstall
```

The uninstaller removes the queue and files installed by this project. It does
not automatically restore the CUPS sandbox configuration because the installer
creates backups and multiple package versions may have been installed on the
same machine. Backups are stored under:

```text
/private/etc/cups/cups-files.conf.hp1136xqx-backup-*
```
